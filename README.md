# Assignment Gallery

## Architecture

I used **MVVM Architecture** with **SOLID Principles** to build this application.

I tried to follow Clean Architecture and keep the code simple and easy to understand.

---

## Libraries Used

- Kotlin
- Jetpack Compose
- Hilt
- Retrofit
- Room Database
- Coil
- Coroutines
- StateFlow
- MockK
- JUnit

---

## Features

- Photo List
- Search by Title
- Search by Photo ID
- Pagination
- Photo Detail
- Favorite using Room Database
- Create Post
- Optional Image Upload
- Unit Testing for Repository and ViewModel

---

# Trade-offs / Future Improvements

Due to the time constraint, I focused on completing the required functionality first.

If I had more time, I would improve the project in the following areas:

1. I used **MVVM** and tried to follow **SOLID Principles**, but there are still a few places where I can improve the code to follow SOLID more completely.

2. I would improve the UI to make it more user-friendly and polished with better animations, spacing, and responsiveness.

3. I would save the created posts in **Room Database** instead of keeping them only in memory. I would also create a separate screen to display all created posts and allow users to view full post details using navigation.

4. I would increase the test coverage by writing tests for the complete application, including UseCases, Repository, ViewModels, and Compose UI tests.

5. I would create more reusable UI components and common utility classes to make the project cleaner, easier to maintain, and easier to scale.

6. I would improve error handling and add better loading states for all network operations.

7. I would add offline support and data caching so the application can still work without an internet connection.
